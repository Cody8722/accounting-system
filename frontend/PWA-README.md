# 💰 PWA 完整版部署指南

## 📱 什麼是 PWA？

PWA (Progressive Web App) 是可以「安裝」到手機主畫面的網頁應用，使用起來就像原生 App！

本記帳系統的 PWA 版本具備：
- ✅ 可安裝到手機主畫面
- ✅ 全螢幕體驗（沒有瀏覽器 UI）
- ✅ **離線功能**：無網路時可以查看已載入的記錄
- ✅ **離線新增**：沒有網路時新增的記錄會暫存
- ✅ **自動同步**：網路恢復後自動同步記錄到伺服器
- ✅ **自動更新**：有新版本時自動更新

## 🚀 部署步驟

### 步驟 1：生成 App 圖示

1. 用瀏覽器打開 `icon-generator.html` 檔案
2. 點擊「下載 icon-192.png」按鈕
3. 點擊「下載 icon-512.png」按鈕
4. 將下載的兩個檔案放到 `frontend/` 資料夾

**或者**，您也可以使用自己設計的圖示（必須是 PNG 格式，尺寸分別為 192x192 和 512x512）

### 步驟 2：提交並推送到 Zeabur

```bash
git add .
git commit -m "Add PWA support with offline capabilities"
git push
```

### 步驟 3：等待 Zeabur 部署

Zeabur 會自動部署，約 1-2 分鐘後即可使用。

## 📱 安裝到手機

### Android (Chrome)

1. 用 Chrome 打開 `https://accounting-system.zeabur.app`
2. 瀏覽器會自動提示「安裝應用程式」
3. 點擊「安裝」或「加入主畫面」
4. 完成！手機桌面會出現「記帳本」圖示

**如果沒有自動提示**：
1. 點擊瀏覽器右上角選單（三個點）
2. 選擇「安裝應用程式」或「加入主畫面」

### iOS (Safari)

1. 用 Safari 打開 `https://accounting-system.zeabur.app`
2. 點擊底部的「分享」按鈕（方框加向上箭頭）
3. 滑動找到「加入主畫面」
4. 點擊「新增」
5. 完成！主畫面會出現「記帳本」圖示

### 桌面電腦 (Chrome/Edge)

1. 打開網站後，網址列右側會出現「安裝」圖示
2. 點擊安裝圖示
3. 確認安裝
4. 完成！可以從桌面或開始選單啟動

## 🌐 離線功能使用說明

### 離線查看記錄

1. 當有網路時，正常瀏覽記錄
2. 記錄會自動快取
3. 沒有網路時，仍可查看之前載入過的記錄

### 離線新增記錄

1. 沒有網路時，仍可新增記錄
2. 新增後會顯示「記錄已儲存，將在連線後同步」
3. 記錄暫時儲存在手機本地
4. 網路恢復後，記錄會**自動同步**到伺服器

### 同步狀態通知

- 📴 網路斷線時：右上角顯示橘色通知「離線模式：記錄將在連線後同步」
- 📡 網路恢復時：右上角顯示綠色通知「已連線，正在同步記錄...」
- ✅ 同步完成後：顯示「離線記錄已同步」

## 🔧 技術細節

### 快取策略

- **靜態資源** (HTML, CSS, JS, 圖示)：Cache First（優先使用快取）
- **API 請求** (查詢記錄)：Network First（優先使用網路，失敗才用快取）
- **寫入請求** (新增/修改/刪除)：離線時暫存，連線後同步

### 離線儲存

- 使用 **IndexedDB** 儲存離線新增的記錄
- 使用 **Cache API** 快取靜態資源和 API 回應
- 資料完全儲存在本地，不依賴網路

### 自動同步

- 使用 **Background Sync API**（如果瀏覽器支援）
- 網路恢復時自動觸發同步
- 手動同步機制作為備援

## ✅ 驗證 PWA 功能

### 檢查 Service Worker

1. 打開 Chrome DevTools (F12)
2. 切換到「Application」分頁
3. 左側選單找到「Service Workers」
4. 應該看到 service-worker.js 狀態為「activated and is running」

### 檢查快取

1. 在 Application 分頁
2. 左側選單找到「Cache Storage」
3. 應該看到「accounting-system-v1」快取
4. 展開可以看到已快取的資源

### 檢查離線佇列

1. 在 Application 分頁
2. 左側選單找到「IndexedDB」
3. 應該看到「AccountingOfflineDB」
4. 展開「offline-queue」可以看到待同步的記錄

### 測試離線功能

1. 打開網站並登入
2. 查看一些記錄（讓它們被快取）
3. 在 DevTools 的 Network 分頁勾選「Offline」
4. 重新整理頁面，應該仍可正常顯示
5. 嘗試新增記錄，應該會提示「將在連線後同步」
6. 取消 Offline 勾選
7. 應該會看到同步通知，記錄已上傳到伺服器

## 🐛 常見問題

### Q: 為什麼沒有「安裝」提示？

A: 確認以下條件：
- 使用 HTTPS（Zeabur 自動提供）
- 有有效的 manifest.json
- 有有效的 Service Worker
- 有 192x192 和 512x512 的圖示

### Q: 離線記錄沒有同步？

A: 檢查：
1. 確認網路已恢復
2. 查看瀏覽器 Console 是否有錯誤訊息
3. 檢查 IndexedDB 中的 offline-queue 是否還有記錄
4. 手動重新整理頁面觸發同步

### Q: 如何更新 PWA？

A: PWA 會自動更新！
1. 部署新版本到 Zeabur
2. 用戶下次打開 PWA 時會自動檢查更新
3. 如果有新版本，會提示「有新版本可用」
4. 點擊確定後自動更新

### Q: 如何卸載 PWA？

A:
- **Android**: 長按 App 圖示 → 解除安裝
- **iOS**: 長按圖示 → 刪除 App
- **桌面**: 在 App 視窗右上角選單 → 解除安裝

## 📊 效能與限制

### 儲存限制

- 快取大小：依瀏覽器而定，通常至少 50MB
- IndexedDB：依瀏覽器而定，通常至少 50MB
- 對於記帳應用來說，空間綽綽有餘

### 離線功能限制

- ❌ 無法查詢未快取的記錄
- ❌ 無法載入圖表（需要統計 API）
- ✅ 可以查看已快取的記錄
- ✅ 可以新增記錄（會在連線後同步）
- ✅ 可以使用所有 UI 功能
